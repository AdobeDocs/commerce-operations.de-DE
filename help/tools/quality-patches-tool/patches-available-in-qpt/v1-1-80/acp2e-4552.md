# ACP2E-4552: GraphQL-Antwort gibt den Unternehmensstatus nicht zurück

Mit dem Patch ACP2E-4552 wird das Problem behoben, dass der Unternehmensstatus in der GraphQL-Antwort nicht zurückgegeben wird. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 installiert ist. Die Patch-ID lautet ACP2E-4552. Dieses Problem wird voraussichtlich in Adobe Commerce B2B 1.5.2-p3 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) B2B 1.5.2-p3

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) B2B >=1.5.0 &lt;1.5.3

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Die GraphQL-Antwort gibt nicht den Unternehmensstatus zurück.

<u>Voraussetzungen</u>:

[!DNL B2B] ist installiert.

<u>Schritte zur Reproduktion</u>:

1. Wechseln Sie in Admin zu **[!UICONTROL Stores]** > *[!UICONTROL Settings]* > **[!UICONTROL Configuration]** > **[!UICONTROL General]** > **[!UICONTROL B2B Features]** und **[!UICONTROL Enable Company]** auf `Yes`.
1. Melden Sie sich bei Commerce Admin als Admin an.
1. Erstellen Sie drei Firmen mit diesem Status: Aktiv, Abgelehnt und Genehmigung ausstehend.
1. Navigieren Sie in der Admin-Liste zu **[!UICONTROL Customers]** > **[!UICONTROL All Customers]**. Öffnen Sie ein Kundenkonto und weisen Sie in der Liste **[!UICONTROL Associate to Company]** den Kunden den einzelnen erstellten Unternehmen zu.
1. Senden Sie die folgende GraphQL-Anfrage:

   ```
   {
     customer {
       firstname
       lastname
       email
       status
       companies {
        items {
           id
           legal_name
           name
           status
        }
       }
     }
   }
   ```

<u>Erwartete Ergebnisse</u>:

Die GraphQL-Antwort gibt die richtigen Statuswerte (z. B. „Aktiv“, „Ausstehende Genehmigung“, „Abgelehnt“ und „Gesperrt„) für alle mit dem Kunden verbundenen Unternehmen zurück. Die Antwort kann auch gefiltert oder nach Unternehmensstatus sortiert werden.

<u>Tatsächliche Ergebnisse</u>:

Die GraphQL-Antwort enthält nicht die Werte des Unternehmensstatus für Unternehmen, die mit dem Kunden verknüpft sind.
