# ACP2E-4653: Bedingungen für den Kategoriebereich der Warenkorbpreisregel fehlen in [!DNL REST] API

Mit dem Patch ACP2E-4653 wird das Problem behoben, dass der Attributbereich Warenkorbpreisregel für **[!UICONTROL Category (Parent Only)]** und **[!UICONTROL Category (Children Only)]** nicht offen gelegt wird, wenn Regeln über die [!DNL REST]-API abgerufen oder aktualisiert werden. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 installiert ist. Die Patch-ID lautet ACP2E-4653. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.8-p2

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.8 - 2.4.8-p5

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=de). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Wenn Regeln zum Warenkorbpreis über die [!DNL REST]-API abgerufen oder aktualisiert werden, wird der Attributbereich für **[!UICONTROL Category (Parent Only)]** und **[!UICONTROL Category (Children Only)]** nicht offen gelegt.

<u>Schritte zur Reproduktion</u>:

1. Generieren Sie ein Admin-Token, indem Sie eine `POST`-Anfrage an `rest/V1/integration/admin/token` mit den Admin-Anmeldeinformationen senden.
1. Erstellen Sie eine Warenkorb-Preisregel mithilfe der [!DNL REST]-Endpunkt-`rest/V1/salesRules` mit einer Aktionsbedingung, die das `category_ids`-Attribut verwendet.
1. Öffnen Sie im Admin-Bedienfeld die Regel Warenkorb-Preis . Fügen Sie im **[!UICONTROL Actions]** Abschnitt unter **[!UICONTROL Apply the rule only to cart items matching the following conditions (leave blank for all items]** manuell eine **[!UICONTROL Category (Children Only)]** Bedingung hinzu.
1. Legen Sie die Bedingung so fest, dass **[!UICONTROL Category (Children Only)]** nicht `<category_id>` wird.
1. Rufen Sie die Regel über die API ab, indem Sie eine `GET` an `rest/V1/salesRules/<rule_id>` senden.
1. Beobachten Sie die in der Antwort zurückgegebenen `action_condition`.

<u>Erwartete Ergebnisse</u>:

1. Es sollte möglich sein, eine Warenkorb-Preisregel mit der **[!UICONTROL Category (Children Only)]**-Bedingung über die [!DNL REST]-API zu erstellen, abzurufen und zu aktualisieren.
1. Der Attributbereich sollte explizit verfügbar gemacht werden, damit Kategorie, **[!UICONTROL Category (Parent Only)]** und **[!UICONTROL Category (Children Only)]** korrekt unterschieden werden können. Beispielsweise sollte die `action_condition` wie folgt zurückgegeben werden:

   ```
           "action_condition": {
               "condition_type": "Magento\\SalesRule\\Model\\Rule\\Condition\\Product\\Combine",
               "conditions": [
                   {
                       "condition_type": "Magento\\SalesRule\\Model\\Rule\\Condition\\Product",
                       "operator": "!=",
                       "attribute_name": "category_ids",
                       "value": "4",
                       "extension_attributes": {
                           "attribute_scope": ""
                       }
                   },
                   {
                       "condition_type": "Magento\\SalesRule\\Model\\Rule\\Condition\\Product",
                       "operator": "==",
                       "attribute_name": "category_ids",
                       "value": "5",
                       "extension_attributes": {
                           "attribute_scope": "children"
                       }
                   }
               ],
               "aggregator_type": "all",
               "operator": null,
               "value": "1"
           },
   ```

<u>Tatsächliche Ergebnisse</u>:

1. Beide Bedingungsattributnamen werden als `category_ids` zurückgegeben.
1. Es ist nicht klar, wie eine Regel mit der **[!UICONTROL Category (Children Only)]** Bedingung über die [!DNL REST]-API erstellt oder aktualisiert werden soll.
