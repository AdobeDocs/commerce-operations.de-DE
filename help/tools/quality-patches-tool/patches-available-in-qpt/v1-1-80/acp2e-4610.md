# ACP2E-4610: Langsame Ausführung von `sales_clean_quotes` Cron

Der Patch ACP2E-4610 behebt das Problem, dass der `sales_clean_quotes` Cron-Auftrag Leistungsprobleme aufweist. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 installiert ist. Die Patch-ID lautet ACP2E-4610. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.6-p13

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.5 - 2.4.8-p5

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Der `sales_clean_quotes` Cron-Auftrag weist Leistungsprobleme auf.

<u>Schritte zur Reproduktion</u>:

1. Generieren Sie mehrere abgelaufene Anführungszeichen in der Angebotstabelle.
1. Führen Sie den `sales_clean_quotes` Cron-Auftrag aus und messen Sie die Ausführungszeit.

<u>Erwartete Ergebnisse</u>:

Der `sales_clean_quotes` Cron-Job endet innerhalb einer angemessenen Zeit.

<u>Tatsächliche Ergebnisse</u>:

Der `sales_clean_quotes` Cron-Job dauert lange, bis er fertig ist.
