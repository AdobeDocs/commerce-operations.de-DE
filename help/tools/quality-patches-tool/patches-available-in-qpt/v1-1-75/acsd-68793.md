# ACSD-68793: Gültige Produkte werden bei der freigegebenen Katalogzuweisung fälschlicherweise abgelehnt

Mit dem Patch ACSD-68793 wird das Problem behoben, dass gültige Produkte bei der freigegebenen Katalogzuweisung fälschlicherweise zurückgewiesen werden. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.75 installiert ist. Die Patch-ID ist ACSD-68793. Dieses Problem wird voraussichtlich in Adobe Commerce [!DNL B2B] 1.5.3 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) [!DNL B2B] 1.5.2-p1

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) [!DNL B2B] 1.5.1 - 1.5.2-p3

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=de). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Gültige Produkte werden bei der freigegebenen Katalogzuweisung über die [!DNL REST]-API fälschlicherweise zurückgewiesen und das System gibt einen Fehler zurück, der besagt, dass keine vorhandenen SKUs vorhanden sind.

<u>Schritte zur Reproduktion</u>:

1. Erstellen Sie mehrere Produkte.
1. Navigieren Sie in der **[!UICONTROL Admin]** Seitenleiste zu **[!UICONTROL Catalog]** > **[!UICONTROL Shared Catalog]**.
1. Klicken Sie auf **[!UICONTROL Add Shared Catalog]** und erstellen Sie einen freigegebenen Katalog.
1. Weisen Sie alle Produkte dem von Ihnen erstellten freigegebenen Katalog zu.
1. Senden Sie eine [!DNL REST]-API-Anfrage an `/rest/V1/sharedCatalog/3/assignProducts` mit der Liste der SKUs für vorhandene Produkte, um sie dem freigegebenen Katalog zuzuweisen.

<u>Erwartete Ergebnisse</u>:

Eine erfolgreiche Antwort wird zurückgegeben. Fehler sollten nur für Produkte zurückgegeben werden, die nicht vorhanden sind.

<u>Tatsächliche Ergebnisse</u>:

Das System gibt den Fehler *Angefordertes Produkt existiert nicht: %sku.* für eine vorhandene SKU.
