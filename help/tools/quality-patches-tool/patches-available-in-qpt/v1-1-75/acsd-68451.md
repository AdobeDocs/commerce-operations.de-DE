# ACSD-68451: Firmenadministratorlinks zu einem nicht verbundenen Unternehmen über mehrere Websites hinweg

Mit dem Patch ACSD-68451 wird das Problem behoben, dass sich ein Unternehmensadministrator auf einer Website anmeldet, auf einer anderen Website ein nicht verbundenes Unternehmen erstellt und fälschlicherweise mit diesem nicht verbundenen Unternehmen verknüpft wird. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.75 installiert ist. Die Patch-ID ist ACSD-68451. Dieses Problem wird voraussichtlich in Adobe Commerce [!DNL B2B] 1.5.3 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) - [!DNL B2B] 1.5.2-p2

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) - [!DNL B2B] 1.5.2-p1 - 1.5.2-p3

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=de). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Bei einem Setup mit mehreren Websites meldet sich ein Unternehmensadministrator bei einer Website an und erstellt eine Firma. Anschließend öffnet der Administrator im selben Browser eine andere Website und erstellt eine andere Firma. Die Plattform verknüpft fälschlicherweise den zweiten Unternehmensadministrator mit dem ersten Unternehmen, obwohl die Unternehmen nicht miteinander verbunden sind.

<u>Voraussetzungen</u>:

Installieren von [!DNL B2B] und Inventar.

<u>Schritte zur Reproduktion</u>:

1. Erstellen Sie eine zusätzliche Website-, Store- und Store-Ansicht.
1. Aktivieren Sie *[!UICONTROL Company]* unter **[!UICONTROL Stores]** > **[!UICONTROL Configuration]** > **[!UICONTROL General]** > **[!UICONTROL B2B Features]**.
1. Rufen Sie die Standard-Website in der Storefront auf.
1. Erstellen Sie eine neue Firma in der Storefront und melden Sie sich als Unternehmensadministrator an.
1. Rufen Sie in derselben Browser-Sitzung die sekundäre Website auf.
1. Erstellen Sie eine neue Firma in der Storefront für die sekundäre Website.
1. Melden Sie sich beim Admin-Bedienfeld an und überprüfen Sie das Kundenraster.

<u>Erwartete Ergebnisse</u>:

Der neue Unternehmensadministrator bzw. die neue Unternehmensadministratorin bleibt nur dem neu erstellten Unternehmen zugewiesen, auch wenn ein anderer Unternehmensadministrator bzw. eine andere Unternehmensadministratorin im selben Browser auf einer anderen Website angemeldet bleibt.

<u>Tatsächliche Ergebnisse</u>:

Der neue Unternehmensadministrator wird neben dem vorgesehenen Unternehmen auch dem anderen Unternehmen als **Firmenbenutzer** zugewiesen.
