# ACSD-68925: GraphQL-Antworten folgen jetzt HTTP-Spezifikationen und geben 4XX für nicht analysierbare, nicht autorisierte oder ungültige Anfragen zurück

Dieser Patch stellt sicher, dass GraphQL jetzt geeignete HTTP-Status-Codes zurückgibt, wenn Anfragen nicht analysierbar, nicht autorisiert oder anderweitig ungültig sind. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.75 installiert ist. Die Patch-ID ist ACSD-68925. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.8-p3

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Zuvor generierten ungültige GraphQL-Anfragen interne Server-Fehler anstelle der richtigen *4XX*-Antworten, die von der GraphQL über HTTP-Spezifikation definiert wurden.

<u>Schritte zur Reproduktion</u>:

Senden Sie eine GraphQL-Anfrage mit einem unbekannten/falschen Filternamen.

```
  {
    cmsBlocks(identifiers: ["home"]) {
    items {
      title
      }
    }
  }
```

<u>Erwartete Ergebnisse</u>:

Ein *4XX*-Antwort-Code wird zurückgegeben, wenn es nicht möglich ist, die Anfrage zu analysieren, die Anfrage nicht autorisiert ist oder ein anderes allgemeines Problem mit der Anfrage vorliegt. Wenn die Anfrage geparst wird und verarbeitet werden kann, sollte ein *200*-Antwort-Code zurückgegeben werden.

<u>Tatsächliche Ergebnisse</u>:

Ein *interner Server-*) wird als Antwort mit folgendem Fehler in der `exception.log` zurückgegeben:

```
report.CRITICAL: LogicException: Config element "FindFilterType" is not declared in GraphQL schema in lib/internal/Magento/Framework/GraphQl/Config.php:57
```
