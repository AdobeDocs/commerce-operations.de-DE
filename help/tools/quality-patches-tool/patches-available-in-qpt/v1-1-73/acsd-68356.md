# ACSD-68356: [!DNL GraphQL] Warenkorbabfrage gibt einen falschen Rabattbetrag für virtuelle Angebote zurück

Der Patch ACSD-68356 behebt das Problem, dass [!DNL GraphQL] Warenkorbabfrage einen falschen Rabattbetrag für virtuelle Angebote zurückgibt. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.73 installiert ist. Die Patch-ID ist ACSD-68356. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.8-p2

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=de). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

[!DNL GraphQL] Warenkorbabfragen geben falsche Rabattbeträge für virtuelle Angebote zurück und wenden Rabatte auf nicht infrage kommende virtuelle Produkte an.

<u>Schritte zur Reproduktion</u>:

1. Erstellen Sie zwei virtuelle Produkte:
   1. `virtual1` SKU
   1. `virtual2` SKU
1. Erstellen Sie eine Warenkorb-Preisregel:
   1. Einstellung so festgelegt, dass sie automatisch ohne Coupon angewendet wird.
   1. Aktionen:
   1. Prozent des Produktpreisrabatts - 20%.
   1. Gilt nur für Artikel im Warenkorb, bei denen die SKU `virtual2` ist.
1. Hinzufügen von Produkten zu einem neuen leeren Angebot über die Storefront:
   1. Zuerst `virtual1` SKU hinzufügen.
   1. `virtual2` SKU als Sekunde hinzufügen.
1. Rufen Sie Details zum Warenkorb mithilfe der folgenden [!DNL GraphQL] ab:

   ```
   {
     cart(cart_id: "cart_id") {
       email
       itemsV2 {
         total_count
         items {
           uid
           prices {
             total_item_discount {
               value
             }
             price {
               value
             }
             discounts {
               label
               amount {
                 value
               }
             }
           }
           product {
             name
             sku
           }
           quantity
         },
         page_info {
             page_size
             current_page
             total_pages
         }
       }
       applied_coupons {
         code
       }
       prices {
         discounts {
           amount {
             value
           }
           applied_to
           label
         }
         grand_total {
           value
         }
       }
     }
   }
   ```

1. Überprüfen Sie **[!UICONTROL cart]** > **[!UICONTROL items]** > **[!UICONTROL prices]** > **[!UICONTROL discounts]**.

<u>Erwartete Ergebnisse</u>:

* `virtual1` SKU hat keinen Rabatt in [!DNL GraphQL] aufgelistet
* `virtual2` SKU hat Rabatt in [!DNL GraphQL] aufgelistet

<u>Tatsächliche Ergebnisse</u>:

* `virtual1` und `virtual2` weisen in der [!DNL GraphQL]-Antwort fälschlicherweise denselben Rabattbetrag auf.
