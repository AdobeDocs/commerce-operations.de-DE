# ACSD-69016: Sonderpreise gelten nicht für Websites mit unterschiedlichen Zeitzonen

Der Patch ACSD-69016 behebt das Problem, dass Produktaktualisierungs- [!DNL REST] API-Anfragen fehlschlagen, wenn kein Speicher mit dem standardmäßigen Code vorhanden ist, was verhindert, dass Sonderpreise für Websites wirksam werden, die in verschiedenen Zeitzonen erstellt wurden. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77 installiert ist. Die Patch-ID ist ACSD-69016. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.8-p1

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.7 - 2.4.8-p3

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Anfragen zur Produktaktualisierung [!DNL REST] API schlagen fehl, wenn kein Speicher mit dem standardmäßigen Code vorhanden ist, was verhindert, dass Sonderpreise für Websites wirksam werden, die in verschiedenen Zeitzonen erstellt wurden.

<u>Schritte zur Reproduktion</u>:

1. Gehen Sie im **[!UICONTROL Admin panel]** zu **[!UICONTROL Stores]** > **[!UICONTROL &#x200B; All Stores]** > **[!UICONTROL Default Store View]**.
1. Ändern Sie den Code-Wert der standardmäßigen Store-Ansicht.
1. Senden Sie eine [!DNL REST] API-Anfrage, um ein Produkt zu aktualisieren.

<u>Erwartete Ergebnisse</u>:

Die Anfrage wurde erfolgreich abgeschlossen und das Produkt wurde aktualisiert.

<u>Tatsächliche Ergebnisse</u>:

Die Anfrage schlägt fehl und das Produkt wird nicht aktualisiert.
