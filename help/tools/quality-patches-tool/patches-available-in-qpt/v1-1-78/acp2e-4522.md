# ACP2E-4522: Zeitweise treten doppelte Schlüsselfehler in der Tabelle quote_coupons auf, wenn mehrere Anfragen zum Zusammenführen des Warenkorbs oder zum Speichern von Angeboten gleichzeitig ausgeführt werden

Mit dem Patch ACP2E-4522 wird das Problem behoben, dass ein zeitweiliger Fehler mit doppelten Schlüsseln in der Tabelle quote_coupons auftritt, wenn mehrere Anfragen zum Zusammenführen des Warenkorbs oder zum Speichern von Angeboten gleichzeitig ausgeführt werden. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 installiert ist. Die Patch-ID lautet ACP2E-4522. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.7-p7

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.7 - 2.4.8-p4

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Ein zeitweiliger Fehler mit doppelten Schlüsseln tritt in der `quote_coupons` auf, wenn mehrere Anfragen zum Zusammenführen des Warenkorbs oder zum Speichern von Angeboten gleichzeitig ausgeführt werden.

<u>Schritte zur Reproduktion</u>:

1. Aktivieren Sie die maximale Anzahl von Coupons pro Bestellung = 10 in der Admin-Seitenleiste unter **[!UICONTROL Stores]** > **[!UICONTROL Settings]** > **[!UICONTROL Configuration]** > **[!UICONTROL Sales]** > **[!UICONTROL Multicoupon Settings]**.
1. Erstellen Sie eine Warenkorb-Preisregel unter **[!UICONTROL Marketing]** > **[!UICONTROL Promotions]** > **[!UICONTROL Cart Price Rules]** > **[!UICONTROL  Add New Rule]**:
   1. **[!UICONTROL Rule Name]**: TESTCOUPON2026
   1. **[!UICONTROL Active]**: **[!UICONTROL Yes]**; **[!UICONTROL Websites]**: **[!UICONTROL Main Website]**; **[!UICONTROL Customer Groups]**; Alle anwendbaren Gruppen auswählen.
   1. **[!UICONTROL Coupon]**: **[!UICONTROL Specific Coupon]**; **[!UICONTROL Coupon Code]**: TESTCOUPON2026.
   1. **[!UICONTROL Uses per Customer]**: Leer lassen.
   1. **[!UICONTROL Conditions]**: Keine Bedingungen hinzugefügt.
   1. **[!UICONTROL Actions]** > **[!UICONTROL Percent of product price discount]**; **[!UICONTROL Discount Amount]** = 10.
1. Fügen Sie mit GraphQL ein Produkt mit einer Mindestmenge von 7 in einen Gastwarenkorb hinzu und verwenden Sie dann den Coupon.
1. Führen Sie sechs gleichzeitige GraphQL-MergeCarts-Aufrufe aus:

   ```
   mutation {
     mergeCarts(
       source_cart_id: "<GUEST_CART_ID>"
       destination_cart_id: "<DESTINATION_CART_ID>"
     ) {
       id
       items { id product { sku } }
       applied_coupons { code }
     }
   }
   ```

<u>Erwartete Ergebnisse</u>:

Bei der GraphQL-Zusammenführung wird kein Fehler zurückgegeben.


<u>Tatsächliche Ergebnisse</u>:

Die GraphQL-Mutation „mergeCarts“ gibt einen Fehler zurück.
Protokolle zeigen, dass der GraphQL-Fehler durch eine Verletzung der Schlüsseleinschränkung in der `quote_coupons`-Tabelle (QUOTE_COUPONS_QUOTE_ID_COUPON_CODE) verursacht wird, wenn mehrere Zusammenführungs- oder Zitatspeicheranfragen gleichzeitig ausgeführt werden.

```
tail -n 200 var/log/exception.log var/log/debug.log | grep -E "Duplicate entry|QUOTE_COUPONS_QUOTE_ID_COUPON_CODE"
```
