# ACP2E-4613: Großes Medienverzeichnis führt zu Verzögerungen beim Laden der Media Gallery-Verzeichnisstruktur

Der Patch ACP2E-4613 behebt das Problem, dass große Medienverzeichnisstrukturen langsame `gettree`-Antworten verursachen, was zu verlängerten Ladezeiten für die **[!UICONTROL Media Gallery]** Verzeichnisstruktur führt. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 installiert ist. Die Patch-ID lautet ACP2E-4613. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.8-p3

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.6 - 2.4.8-p4

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=de). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Große Medienverzeichnisstrukturen führen zu langsamen `gettree`, was zu verlängerten Ladezeiten für die [!UICONTROL Media Gallery] führt.

<u>Voraussetzungen</u>:

1. Es gibt eine große Anzahl von Produkten mit mehreren Bildern.

<u>Schritte zur Reproduktion</u>:

1. Generieren von Miniaturansichten, um den Medienspeicherbedarf zu erhöhen:

   ```
   bin/magento catalog:images:resize
   ```

1. Erstellen Sie mehrere Unterverzeichnisse unter `pub/media/catalog/category` (z. B. test, test2, test3, test4 und ggf. zusätzliche verschachtelte Ordner).
1. Kopieren Sie einen großen Satz an Katalogmediendateien in die neu erstellten Verzeichnisse, um die Größe der Medienverzeichnisstruktur erheblich zu erhöhen, z. B.:

   ```
   cp -r pub/media/catalog/product/* pub/media/catalog/category/test/
   cp -r pub/media/catalog/product/* pub/media/catalog/category/test2/
   (...)
   ```

1. Melden Sie sich beim Admin-Bedienfeld an und navigieren Sie zu **[!UICONTROL Content]** > *[!UICONTROL Media]* > **[!UICONTROL Media Gallery]**.
1. Beachten Sie, dass der Abschluss der `admin/media_gallery/directories/gettree/?isAjax=true`-Anfrage viel Zeit in Anspruch nimmt.

<u>Erwartete Ergebnisse</u>:

Die Verzeichnisstruktur wird innerhalb weniger Sekunden geladen.

<u>Tatsächliche Ergebnisse</u>:

Das Laden der Verzeichnisstruktur dauert abhängig von der Anzahl der Unterverzeichnisse unter `pub/media` mehrere Minuten.
