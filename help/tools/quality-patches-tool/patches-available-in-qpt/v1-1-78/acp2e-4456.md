# ACP2E-4456: Durch Stornieren einer Bestellung mit einer GraphQL-Mutation werden Bestellungen, die vollständig mit Geschenkkarten bezahlt wurden, nicht in den Status „Geschlossen“ verschoben

Mit dem Patch ACP2E-4456 wird das Problem behoben, dass bei der Stornierung von GraphQL-Bestellungen keine per Geschenkkarte bezahlten Bestellungen in den [!UICONTROL Closed] übergegangen sind. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 installiert ist. Die Patch-ID lautet ACP2E-4456. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.8-p3

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=de). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Ein Problem tritt auf, wenn beim Stornieren einer Bestellung über eine GraphQL-Mutation die mit der Geschenkkarte bezahlten Bestellungen nicht in den [!UICONTROL Closed] übergehen.

<u>Schritte zur Reproduktion</u>:

1. Erstellen Sie eine Geschenkkarte beliebigen Typs.
1. Erstellen Sie ein einfaches Produkt.
1. **[!UICONTROL Free Shipping]** aktivieren.
1. Aktivieren Sie die Stornierung von GraphQL-Bestellungen, indem Sie zu **[!UICONTROL Stores]** > **[!UICONTROL Configuration]** > **[!UICONTROL Sales]** > **[!UICONTROL Order cancellation]** navigieren und **[!UICONTROL Order cancellation through GraphQL]** auf *Ja* setzen.
1. Beim Storefront-Konto anmelden.
1. Kaufen Sie eine Geschenkkarte, generieren Sie eine Rechnung für die Bestellung und notieren Sie sich die Geschenkkartennummer.
1. Fügen Sie das einfache Produkt zum Warenkorb hinzu.
1. Wählen Sie **[!UICONTROL Free Shipping]** während des Checkouts aus.
1. Füllen Sie die Zahlung mit der Geschenkkarte aus.
1. Rechnung der Bestellung.
1. Brechen Sie die Bestellung mit der folgenden GraphQL-Mutation ab:

   ```
   mutation {
   	cancelOrder(input: { order_id: "Nw==", reason: "Other" }) {
   		error
   		order {
   			carrier
   			created_at
   			email
   			gift_receipt_included
   			grand_total
   			id
   			increment_id
   			number
   			order_date
   			shipping_method
   			status
   		}
   	}
   }
   ```

<u>Erwartete Ergebnisse</u>:

Der Bestellstatus ändert sich in Storniert.

<u>Tatsächliche Ergebnisse</u>:

Die Bestellung wird nicht storniert.
