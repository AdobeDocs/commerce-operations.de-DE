# ACP2E-4431: [!UICONTROL Related Products], denen Zielregeln zugeordnet sind, werden während des Neuindizierungsprozesses gelöscht

Mit dem Patch ACP2E-4431 wird das Problem behoben, dass die von den Zielregeln übereinstimmenden **[!UICONTROL Related Products]** während des Neuindizierungsprozesses gelöscht werden. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 installiert ist. Die Patch-ID lautet ACP2E-4431. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.7-p4

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.4 - 2.4.8-p4

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=de). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Die **[!UICONTROL Related Products]**, denen die Zielregeln entsprechen, werden während des Neuindizierungsprozesses gelöscht.

<u>Schritte zur Reproduktion</u>:

1. Erstellen Sie 10000 oder mehr Produkte.
1. Weisen Sie mindestens 10000 Produkten mindestens 5.000 Kategorien zu.
1. Erstellen Sie 100 oder mehr verwandte Produktregeln.
1. Konfigurieren Sie Regelbedingungen, bei denen die **[!UICONTROL Products]**, die mit der **[!UICONTROL Product Category]** übereinstimmen soll, *X*.
1. Die mit **[!UICONTROL Product Category]** anzuzeigende **[!UICONTROL Products]** ist **[!UICONTROL Constant Value]** *Y*.
1. Führen Sie diese GraphQL-Abfrage aus, um `related_products` für ein Produkt abzurufen:

   ```graphql
   {
       products(filter: { url_key: { eq: "simple-product-491" } }) {
           items {
               sku
               related_products {
                   uid
                   sku
                   name
                   url_key
                   stock_status
                   price_range {
                       minimum_price {
                           regular_price {
                              value
                           }
                           final_price {
                               value
                           }
                       }
                   }
               }
           }
       }
   }
   ```

1. Erstellen Sie einen großen Rückstand für den `targetrule_product_rule`.
1. Erstellen Sie einen großen Rückstand für den `targetrule_rule_product`.
1. Führen Sie den `targetrule_rule_product` aus.
1. Führen Sie den `targetrule_product_rule` aus.

<u>Erwartete Ergebnisse</u>:

Die `related_products` enthält immer Produkte, die den Regelbedingungen entsprechen.

<u>Tatsächliche Ergebnisse</u>:

Die `related_products` gibt ein leeres -Array zurück, während die Indexer ausgeführt werden.
