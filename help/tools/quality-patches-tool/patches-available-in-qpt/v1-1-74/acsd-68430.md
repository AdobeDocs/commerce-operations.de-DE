# ACSD-68430: Speichern einer Kunden- oder Kundenadresse fehlgeschlagen, wenn der Datensatz mehrere Attributoptionen enthielt, die aus der Attributkonfiguration gelöscht wurden

Mit dem Patch ACSD-68430 wird ein Problem behoben, bei dem das Speichern einer Kunden- oder Kundenadresse fehlschlägt, wenn der Datensatz mehrere Attributoptionen enthält, die aus der Attributkonfiguration gelöscht wurden. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.74 installiert ist. Die Patch-ID ist ACSD-68430. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.7-p6

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.6 - 2.4.7-p8

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=de). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Beim Speichern einer Kunden- oder Kundenadresse, die mehrere Attributoptionen enthält, die aus der Attributkonfiguration gelöscht wurden, schlägt der Vorgang fehl.

<u>Schritte zur Reproduktion</u>:

1. Navigieren Sie zu Admin und erstellen Sie ein benutzerdefiniertes Kundenattribut mit mehreren Auswahlmöglichkeiten (Beispiel: „test„) mit den Optionen A, B, C, D.
1. Erstellen Sie einen Kunden aus Admin und speichern Sie Werte für dieses Attribut (Beispiel: Wählen Sie A, B, C).
1. Gehen Sie zurück zur Attributkonfiguration und löschen Sie eine der Optionen, die beim Kunden gespeichert wurden (Beispiel: Löschoption C).
1. Versuchen Sie in Admin, den Datensatz desselben Kunden zu bearbeiten - ändern Sie ein beliebiges Feld (wie Geschlecht oder Präfix) und klicken Sie auf **[!UICONTROL Save]**.

<u>Erwartete Ergebnisse</u>:

Änderungen an Kundendatensätzen werden fehlerfrei gespeichert.

<u>Tatsächliche Ergebnisse</u>:

Beim Speichern von Änderungen am Kundendatensatz tritt ein Fehler auf: *Attribut [attribute_name] enthält keine Option mit der ID [delete_option_id]*.
