# ACSD-69129: [!DNL REST] Aktualisierung der API-Stufe schlägt fehl, nachdem die standardmäßige Basis-Website gelöscht und die sekundäre Website als Standard verwendet wurde

Mit dem Patch ACSD-69129 wird das Problem behoben, dass das Löschen der Standard-Basis-Website und die Verwendung der sekundären Website als Standard zu einem Fehler führte, wenn versucht wurde, den Stufenpreis für die sekundäre Website über [!DNL REST] API zu aktualisieren. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 installiert ist. Die Patch-ID ist ACSD-69129. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.6-p13, 2.4.8-p3

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.5 - 2.4.6-p13, 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Das Löschen der Standard-Basis-Website und die Verwendung der sekundären Website als Standard führen zu einem Fehler, wenn versucht wird, den Stufenpreis für die sekundäre Website über [!DNL REST] API zu aktualisieren.

<u>Schritte zur Reproduktion</u>:

1. Melden Sie sich bei [!UICONTROL Admin] Panel an.
1. **[!UICONTROL Catalog Price Scope]** = *Website*.
1. Erstellen Sie eine neue Website/einen neuen Store/eine neue Storeview.
1. Legen *Als Standard festlegen* für die neu erstellte Website fest.
1. Löschen Sie den Cache.
1. Erstellen Sie ein neues Produkt. Weisen Sie sie nur der zweiten Website zu.
1. Senden Sie eine [!DNL REST] API-Anfrage, um den Stufenpreis für die zweite Website festzulegen.
1. Das Ergebnis sollte ein leeres Array mit einer *200*-Antwort sein.
1. Löschen Sie in der [!UICONTROL Admin] *Haupt-Website*.
1. Senden Sie eine [!DNL REST] API-Anfrage, um den Stufenpreis für die zweite Website festzulegen.

<u>Erwartete Ergebnisse</u>:

* Eine *200*-Antwort wird empfangen.
* Der Stufenpreis für das Produkt wird aktualisiert.

<u>Tatsächliche Ergebnisse</u>:

* Das *Ungültiges Attribut Website-ID = %websiteId. Zeilen-ID: SKU = %SKU, Website-ID: %websiteId, Kundengruppe: %customerGroup, Menge: %quantity.* Antwort wird ein Fehler zurückgegeben.
* Der Stufenpreis für das Produkt wird nicht aktualisiert.
