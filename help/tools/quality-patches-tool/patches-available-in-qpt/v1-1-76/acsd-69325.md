# ACSD-69325: Durch Änderung des SKU-Falls erscheint das Produkt nicht mehr vorrätig in der Storefront

Mit dem Patch ACSD-69325 wird das Problem behoben, dass beim Ändern einer Produkt-SKU zur Verwendung von Kleinbuchstaben das Produkt als nicht vorrätig auf der Storefront angezeigt wird. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 installiert ist. Die Patch-ID ist ACSD-69325. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.8-p3

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.7 - 2.4.8-p3

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=de). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Wenn die SKU so geändert wird, dass sie Kleinbuchstaben enthält, wird das Produkt in der Storefront fälschlicherweise als nicht vorrätig angezeigt.

<u>Schritte zur Reproduktion</u>:

1. Melden Sie sich beim Admin Panel an.
1. Erstellen Sie eine zusätzliche Website-, Store- und Store-Ansicht, indem Sie zu **[!UICONTROL Stores]** > **[!UICONTROL All Stores]** navigieren.
1. Erstellen Sie einen Vertriebskanal für die neue Website, indem Sie zu **[!UICONTROL Channels]** > **[!UICONTROL Sales Channels]** navigieren.
1. Erstellen Sie ein Produkt, das der neuen Website mit SKU = &#39;RF-42R-80X100-CL&#39; zugewiesen ist, und weisen Sie es dem Vertriebskanal zu.
1. Führen Sie die Indexer durch Ausführen von `php bin/magento indexer:reindex` aus und stellen Sie sicher, dass geplante Aufgaben (cron) ausgeführt werden.
1. Öffnen Sie die Produktseite in der Storefront.
1. Überprüfen Sie, ob das Produkt *Auf Lager* ist.
1. Aktualisieren Sie die SKU auf `RF-42R-80x100-CL` (mit einem **(x**).
1. Führen Sie die Indexer durch Ausführen von `php bin/magento indexer:reindex` aus und stellen Sie sicher, dass geplante Aufgaben (cron) ausgeführt werden.
1. Öffnen Sie die Produktseite in der Storefront.
1. Überprüfen Sie den Status des Produktbestands.

<u>Erwartete Ergebnisse</u>:

Das Produkt verbleibt *Auf Lager* in der Storefront.

<u>Tatsächliche Ergebnisse</u>:

Das Produkt wird als *Nicht vorrätig* in der Storefront angezeigt.
