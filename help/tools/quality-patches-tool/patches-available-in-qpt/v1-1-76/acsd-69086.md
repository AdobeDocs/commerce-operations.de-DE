# ACSD-69086: Installation schlägt auf MariaDB 10.11 aufgrund einer nicht unterstützten Datenbankversionsprüfung fehl

Der Patch ACSD-69086 behebt das Problem, dass die Installation der Adobe Commerce-Instanz bei Verwendung von MariaDB 10.11 fehlgeschlagen ist. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 installiert ist. Die Patch-ID ist ACSD-69086. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.7-p8

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.7 - 2.4.7-p8

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=de). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Die Installation der Adobe Commerce-Instanz schlägt bei Verwendung von MariaDB 10.11 fehl.

<u>Schritte zur Reproduktion</u>:

1. Installieren Sie Adobe Commerce auf MariaDB Version 10.11.
1. Führen Sie den standardmäßigen Installationsprozess aus.
1. Führen Sie `setup:upgrade` nach der Installation aus.

<u>Erwartete Ergebnisse</u>:

Installation und `setup:upgrade` erfolgreich abgeschlossen auf MariaDB Version 10.11.

<u>Tatsächliche Ergebnisse</u>:

1. Die Installation auf MariaDB 10.11 schlägt während der Datenbankschema-Installation fehl.
1. Der Prozess wird mit einem Fehler beendet *Aktuelle Version von RDBMS wird nicht unterstützt*.

## Patch anwenden

Verwenden Sie je nach Bereitstellungsmethode die folgenden Links, um einzelne Patches anzuwenden:

* Adobe Commerce oder Magento Open Source On-Premise: [[!DNL Quality Patches Tool] > Nutzung](/help/tools/quality-patches-tool/usage.md) im [!DNL Quality Patches Tool].
* Adobe Commerce in Cloud-Infrastruktur: [Upgrades und Patches > Patches anwenden](https://experienceleague.adobe.com/docs/commerce-cloud-service/user-guide/develop/upgrade/apply-patches.html?lang=de) im Handbuch zu Commerce in Cloud-Infrastruktur.
