# ACP2E-4601: Erneute Speicherung einer Zahlungsautorisierungstransaktion `parent_id` Selbstverweis und Trigger einer Endlosschleife/OOM

Mit dem Patch ACP2E-4601 wird das Problem behoben, dass sich die Verarbeitung von Zahlungsvorgängen beim erneuten Speichern von Autorisierungstransaktionen ohne `parent_txn_id` ineffizient verhält. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79 installiert ist. Die Patch-ID lautet ACP2E-4601. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.8-p3

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=de). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Die Verarbeitung von Zahlungstransaktionen verhält sich beim erneuten Speichern von Autorisierungstransaktionen, die ohne `parent_txn_id` erstellt wurden, ineffizient.

<u>Schritte zur Reproduktion</u>:

1. Richten Sie eine Online-Zahlungsmethode ein.
1. Bestellung, durch die eine Autorisierungs-Zahlungsbuchung ohne `parent_txn_id` erstellt wird.
1. Speichern Sie die Autorisierungstransaktion erneut.

<u>Erwartete Ergebnisse</u>:

Die Verarbeitung der Zahlungstransaktion wurde erfolgreich abgeschlossen.

<u>Tatsächliche Ergebnisse</u>:

Das Speichern der Autorisierungstransaktion schlägt fehl.
