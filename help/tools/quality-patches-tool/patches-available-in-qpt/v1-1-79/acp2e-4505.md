# ACP2E-4505: Zirkuläre Kategorieabhängigkeit vom Speichern einer Kategorie mit veralteten Daten in einer doppelten Browser-Registerkarte

Der Patch ACP2E-4505 behebt das Problem, dass es möglich war, eine Kategorie mit veralteten Daten von einer doppelten Browser-Registerkarte zu speichern, wodurch eine zirkuläre Abhängigkeit erzeugt wurde. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79 installiert ist. Die Patch-ID lautet ACP2E-4505. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.8-p3, 2.4.7-p8

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.7 - 2.4.8-p4

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=de). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Es war möglich, eine Kategorie mit veralteten Daten aus einer doppelten Browser-Registerkarte zu speichern, wodurch eine zirkuläre Abhängigkeit erzeugt wurde.

<u>Schritte zur Reproduktion</u>:

1. Erstellen Sie eine Kategoriestruktur: A > B > C.
1. Navigieren Sie im Admin-Bedienfeld zu **[!UICONTROL Catalog]** > **[!UICONTROL Categories]**.
1. Wählen Sie in der Kategoriestruktur Kategorie C aus, um das Bearbeitungsformular in Registerkarte 1 zu öffnen.
1. Duplizieren Sie die Browser-Registerkarte, um Registerkarte 2 mit demselben Bearbeitungsformular für Kategorie C zu öffnen.
1. Registerkarte 1:
   1. Kategorie C unter A ziehen, sodass C ein untergeordnetes Element von A wird.
   1. Ziehen Sie Kategorie B unter C, sodass B ein untergeordnetes Element von C wird.
   1. Warten Sie, bis beide Drag-and-Drop-Vorgänge abgeschlossen sind und die Kategoriestruktur aktualisiert wird.
1. In Registerkarte 2 (zeigt weiterhin die alte Hierarchie an):
   1. Nehmen Sie beliebige Änderungen vor, bearbeiten Sie beispielsweise die Beschreibung.
   1. Klicken Sie auf **[!UICONTROL Save]**.

<u>Erwartete Ergebnisse</u>:

Das System sollte zirkuläre Beziehungen zwischen übergeordneten und untergeordneten Elementen validieren und verhindern, dass sie gespeichert werden.

<u>Tatsächliche Ergebnisse</u>:

Eine zirkuläre hierarchisch untergeordnete Abhängigkeit zwischen Kategorien wird erstellt (Kategorie B hat übergeordnetes C, Kategorie C hat übergeordnetes B), was zu einer unendlichen Rekursion bei der Durchlaufbewegung der Kategoriehierarchie führt.
