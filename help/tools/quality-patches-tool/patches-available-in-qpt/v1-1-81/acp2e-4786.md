# ACP2E-4786: Produktexport schlägt fehl, wenn der Remote-Speicher AWS S3 konfiguriert ist

Mit dem Patch ACP2E-4786 wird das Problem behoben, dass der Produktexport fehlschlägt, da der Speichertreiber einen ungültigen Pfad verwendet, wenn der AWS S3-Remote-Speicher aktiviert ist. Dieser Patch ist verfügbar, wenn [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.81 installiert ist. Die Patch-ID lautet ACP2E-4786. Dieses Problem wird voraussichtlich in Adobe Commerce 2.4.9 behoben.

## Betroffene Produkte und Versionen

**Der Patch wird für die Adobe Commerce-Version erstellt:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.7-p4

**Kompatibel mit Adobe Commerce-Versionen:**

* Adobe Commerce (alle Bereitstellungsmethoden) 2.4.5 - 2.4.9

>[!NOTE]
>
>Der Patch könnte mit neuen [!DNL Quality Patches Tool]-Versionen auch für andere Versionen gelten. Um zu überprüfen, ob der Patch mit Ihrer Adobe Commerce-Version kompatibel ist, aktualisieren Sie das `magento/quality-patches` auf die neueste Version und überprüfen Sie die Kompatibilität auf der Seite [[!DNL Quality Patches Tool]: Nach Patches suchen](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Verwenden Sie die Patch-ID als Suchbegriff, um den Patch zu finden.

## Problem

Der Produktexport schlägt fehl, wenn der AWS S3-Remotespeicher aktiviert ist, da der Speichertreiber ungültig für die Pfadverarbeitung ist.

<u>Schritte zur Reproduktion</u>:

1. Konfigurieren von [!DNL AWS S3] als Remote-Speicher:

   ```
   bin/magento setup:config:set --remote-storage-driver="aws-s3" --remote-storage-bucket="..." --remote-storage-region="..." --remote-storage-key="..." --remote-storage-secret="..." -n
   ```

1. Deaktivieren Sie den Datenbankspeicher für Mediendateien, indem Sie den folgenden Befehl ausführen:

   ```
   bin/magento config:set system/media_storage_configuration/media_database 0
   ```

1. Navigieren Sie in der Admin-Seitenleiste zu **[!UICONTROL System]** > **[!UICONTROL Data Transfer]** > **Exportieren**.
1. Wählen Sie **[!UICONTROL Entity Type]** &quot;**[!UICONTROL Products]**&quot;.
1. Klicken Sie auf **[!UICONTROL Continue]**.

<u>Erwartete Ergebnisse</u>:

Die Exportdatei wurde erfolgreich erstellt.

<u>Tatsächliche Ergebnisse</u>:

1. Die Exportdatei wird nicht erstellt.
1. In den Protokollen wird der Fehler *SignatureDoesNotMatch* angezeigt.
